Let me save you some pain.

When I started making games I did what everyone does: I googled "best game engine for beginners," spent three weeks reading forum arguments about Unity vs Godot vs Unreal, picked one, started a project, abandoned it, switched engines, and repeated that cycle until I hated myself.

Here's what actually matters.

## The Engine Doesn't Matter (At First)

Pick one. Any one. Godot 4 is free, open source, and genuinely good. Unity is dominant in the industry and has endless tutorials. Unreal is incredible for 3D if you can stomach the learning curve.

What matters: **finish something small before you think about any of this.**

The engine decision matters later, when you know what you're building. At the start, you don't know what you're building. You think you do.

## Start With a Clone, Not an Original Idea

Your original idea is a sprawling open-world RPG with procedural generation and a morality system. I know. I've had that idea too.

**Your first game should be:**
- Pong
- Breakout
- A single-room platformer
- A 30-second arcade game

Not because these ideas are exciting. Because you need to learn the *craft* before you work on the *art*. Clones strip away the design problem so you can focus on the technical one.

## The 80% Problem

Here's something nobody tells beginners: every game project will feel "80% done" for about 60% of its total development time.

Polish takes longer than building. Sound effects, UI, menus, save systems, input handling edge cases, juice — all of this is invisible until it's missing, and it adds up.

Ship something broken if you have to. You'll learn more from releasing one bad game than completing zero good ones.

## My Actual Setup

- **Engine:** Godot 4
- **Art:** Aseprite for pixel, Blender for 3D (badly)
- **Audio:** BFXR for sfx, Audacity for editing, FL Studio when I'm feeling fancy
- **Version control:** Git, always Git
- **Distribution:** itch.io to start, Steam when ready

## The Actual Advice

Make something you can finish in a weekend. Finish it. Then make something you can finish in a week. Finish that. Keep going.

The difference between people who make games and people who want to make games is the finishing.

That's it. Everything else is noise.
